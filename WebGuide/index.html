¿Cómo se divide este proyecto?

Tu proyecto Android utiliza Gradle como motor de compilación. Aquí tienes los archivos más importantes que debes conocer:

  • build.gradle.kts / settings.gradle.kts: Scripts en Kotlin que manejan las dependencias (librerías) de tu app.
  • AndroidManifest.xml: El documento principal que declara las pantallas (Activities) de tu sistema y los permisos necesarios (como acceso a Internet).
  • app/src/main/res/ Tus recursos estáticos:
    • layout/: Contiene archivos XML tradicionales para diseñar visualmente la interfaz.
    • values/: Variables como strings.xml (textos) y colors.xml (colores globales).
  • app/src/main/java/... Todo el código lógico en Kotlin. Es aquí donde vinculas interactividad con tu pantalla.

Creación del Proyecto

Criterios en Android Studio y Solución de problemas de Gradle

🛠️ Pasos para crear un Proyecto desde Cero

  • Abre Android Studio y selecciona "New Project".
  • Elegir Plantilla: Para usar Jetpack Compose nativo, elige "Empty Activity". Para empezar con XML tradicional sin código "molesto" generado de Jetpack, selecciona "Empty Views Activity".
  • Nombre y Paquete: Asigna un título a tu app y un "Package Name" o identificador único reverso (ej: com.tuempresa.miapp). Define su identidad mundial.
  • Minimum SDK (Nivel de API Mínimo): Recomendamos escoger "API 24: Android 7.0" para alcanzar el ~95% del mercado de teléfonos actuales gozando de las modernidades de las librerías al mismo tiempo.
  • Lenguaje: Siempre selecciona Kotlin (Java ya fue desplazado para este tipo de apps modernas) y haz click en "Finish".

⚠️ Problemas Comunes y Frustrantes con Gradle

Gradle es la máquina que empaqueta las piezas, descarga el código de terceros y compila tu app. Cuando Gradle da un error "Sync Failed", aplica estos criterios:

  • Conflictos por Falta de Sincronización: Si acabas de editar los archivos build.gradle.kts, presiona siempre el botón en la barra superior llamado "Sync Project with Gradle Files" (Ícono del elefante) antes de intentar correr la app.
  • Caché Corrupta de Android Studio (Bug de "Todo en Rojo"): A veces Android Studio se vuelve loco y marca toda tu sintaxis correcta como errores de "Cannot Resolve Symbol". Arréglalo yendo a la barra de menú superior: File > Invalidate Caches... y seleccionando "Invalidate and Restart". Esto borrará índices corruptos con fuerza bruta y redibujará el entorno con éxito.
  • JDK Incompatible ("Unsupported class file major version..."): Asegúrate que los ajustes del entorno coinciden con el JDK esperado. Ve a File > Settings (o Preferences en Mac) > Build, Execution > Build Tools > Gradle y asegúrate que "Gradle JDK" es la Versión 17 o superior.
  • Paciencia: La primera vez que Android Studio crea un proyecto puede demorar varios minutos mientras descarga internet completo de repositorios Maven en segundo plano. Nunca modifiques el código si la barra inferior sigue mostrando una barra de carga diciendo "Gradle Syncing...".

Configuración Final

Cómo probar tu App y compartirla con el mundo

🚀 Pasos para lanzar el Emulador

  • Requisitos: Asegúrate de instalar "Android Emulator" desde el SDK Manager de Android Studio.
  • Ve al Device Manager (ícono de teléfono en la barra lateral derecha o superior).
  • Haz clic en Create Device, selecciona un modelo de teléfono (por ejemplo, Pixel 8) y descarga una imagen de sistema (recomendado: API 34+).
  • Una vez creado, selecciona el dispositivo en la barra superior junto al botón verde de "Run/Play".
  • Haz clic en el botón verde Play (Run 'app'). Android Studio compilará el proyecto instalando la aplicación automáticamente en el emulador.

📦 Pasos para generar la APK (Instalable física)

  • Opción Rápida (Debug APK): Ideal para probar en tu teléfono personal.
    ➡ Ve al menú superior: Build > Build Bundle(s) / APK(s) > Build APK(s).
    ➡ Al finalizar, Android Studio mostrará un mensaje. Haz clic en "locate" para abrir la carpeta donde está el archivo app-debug.apk. Pasa ese archivo a tu teléfono.
  • Opción Final (Release Signed APK): Para publicar en tiendas oficiales.
    ➡ Ve al menú superior: Build > Generate Signed Bundle / APK.
    ➡ Elige APK, crea un Key Store Path nuevo, establece contraseñas.
    ➡ Selecciona la variante release y dale a Finish.
